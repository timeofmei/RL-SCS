# 项目目标

将 RL-SCS 的游戏规则、数据格式、单位系统对齐**庙算·陆战指挥官**平台规范，使同一套 RL 训练代码可以直接在庙算规则体系下运行。

参考文档来源：`docs/miaosuan/`（离线存档，原始站点 https://wargame.ia.ac.cn/docs/）

## 核心约束

1. **回合制**：不引入实时帧推进。每回合双方同时提交动作列表，环境计算一次裁决结果。
2. **不依赖庙算 SDK**：完全自行实现，仅参考 `docs/miaosuan/` 中的 MD 文档描述，不 import 庙算任何库。

## 背景差异

| 维度 | 当前 SCS 实现 | 目标 |
|------|--------------|------|
| API 框架 | PettingZoo AEC | 保留，动作/观察格式对齐庙算 |
| 动作空间 | `Discrete` 平面整数 | 结构化字典列表（按庙算 action type） |
| 观察空间 | 自定义张量 | 嵌套字典（算子、裁决、分数、合法动作等） |
| 地图 | YAML 配置 + 自定义 hex 网格 | 保留 hex 网格，添加高程/地形/道路/通视数据 |
| 单位系统 | 简单 unit type | 5 大类算子 + sub_type + 武器 ID |
| 推演节奏 | 回合制 | **保持回合制** |
| 阵营标识 | p1/p2 (0/1) | 红(0)/蓝(1)，席位 ID 区分身份 |
| 实现依赖 | — | 纯自行实现 |
